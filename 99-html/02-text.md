# 02. 텍스트를 표현하는 태그들

## 🤔 왜 텍스트용 태그가 여러 개일까?

사용자에게 보여 주는 글자가 전부 "그냥 글자" 는 아닙니다.
어떤 건 **제목** 이고, 어떤 건 **설명 문단** 이고, 어떤 건 **강조된 단어** 죠.

만약 HTML 에 단 하나의 "글자" 태그만 있다면 이런 문제가 생깁니다.

- 브라우저가 어떤 게 제목이고 어떤 게 본문인지 구분할 수 없어서, **크기를 스스로 정할 수 없음.**
- 검색 엔진이 "이 페이지의 핵심 주제는 뭐지?" 를 판단할 수 없음.
- 시각장애인이 쓰는 **스크린 리더** 가 "지금부터 제목을 읽어 드릴게요" 같은 안내를 못 함.

그래서 HTML 은 글자의 **의미(semantic)** 에 따라 다른 태그를 쓰게 합니다.
이게 바로 "semantic HTML" 이라는 말이 여기저기서 나오는 이유예요.

> 🔑 의미에 맞는 태그를 고르는 건 **"귀찮은 공식 절차"** 가 아니라, **코드를 나중에 유지보수할 당신 자신** 과
> **검색 엔진**, **접근성 도구**, **그리고 디자이너/개발자 동료** 에게 보내는 메시지입니다.

---

## 🏷 제목(heading): `<h1>` ~ `<h6>`

**가장 중요한 제목에 `<h1>`** 을 쓰고, 하위 제목으로 갈수록 숫자가 커집니다.

```html
<h1>점심 뭐 먹지</h1>
<h2>오늘의 추천</h2>
<h3>한식</h3>
<h3>중식</h3>
<h2>인기 음식점</h2>
```

### 규칙과 관례

- **한 페이지에 `<h1>` 은 보통 한 개.** (페이지의 대표 제목)
- 숫자를 건너뛰지 않는 게 관례. `<h1>` 다음에 바로 `<h4>` 로 넘어가지 않기.
- 크기가 마음에 안 든다고 제목이 아닌 곳에 `<h1>` 을 쓰면 안 됩니다.
  크기는 **CSS 로 따로 바꾸는** 것이 맞아요.

### 왜 제목을 계층적으로 쓰나?

스크린 리더는 "H 키" 를 누르면 페이지의 모든 제목만 훑어 볼 수 있게 해줍니다.
마치 **목차** 처럼요.
그래서 제목 태그가 계층적으로 쓰여 있지 않으면 **시각장애인 사용자의 탐색이 엉망이 됩니다.**

---

## 📝 문단: `<p>`

"paragraph" 의 약자. **하나의 문단** 을 감싸는 태그입니다.

```html
<p>
  평양 출신의 할머니가 수십 년간 운영해온 비지 전문점 피양콩 할마니.
  두부를 빼지 않은 되비지를 맛볼 수 있는 곳입니다.
</p>
<p>
  딸과 함께 운영하는 이곳에선 맷돌로 직접 간 콩만을 사용합니다.
</p>
```

브라우저는 `<p>` 마다 자동으로 **위아래 여백** 을 만들어 문단을 구분해 줍니다.

### ❗ 자주 하는 실수

`<p>` 안에 또 `<p>` 를 넣거나, `<p>` 안에 `<div>` 를 넣는 건 **금지** 입니다.
문단 안에 문단이 들어가는 건 의미상 말이 안 되니까요.
브라우저가 이런 걸 발견하면 몰래 태그를 끊어 버리는 경우도 있어서 결과가 이상해집니다.

---

## 🔤 줄바꿈과 가로선: `<br>`, `<hr>`

### `<br>` — 한 줄 바꾸기

```html
<p>
  안녕하세요.<br>
  오늘도 좋은 하루 되세요.
</p>
```

**문단 안에서 한 줄만 바꾸고 싶을 때** 씁니다.
주의: `<br>` 을 연속으로 써서 여백을 만들려고 하는 건 **금지**.
여백은 CSS 로 만듭니다.

### `<hr>` — 가로 구분선

```html
<section>이전 내용</section>
<hr>
<section>새 주제</section>
```

주제가 바뀌는 지점에 쓰는 **의미 있는 구분선** 입니다.

---

## ✨ 글자 강조: `<strong>`, `<em>`

글자 일부를 강조할 때 쓰는 태그 두 개가 있어요. 둘은 **의미가 조금 다릅니다.**

```html
<p>
  이번 회의는 <strong>내일 오후 3시</strong> 에 시작합니다.
  <em>꼭</em> 참석해 주세요.
</p>
```

| 태그 | 기본 스타일 | 의미 |
|---|---|---|
| `<strong>` | **굵게** | "이 부분은 중요하다" |
| `<em>` | *기울여서* | "이 부분은 말의 강조점이다" (emphasis) |

### 왜 `<b>`, `<i>` 를 안 쓰고 `<strong>`, `<em>` 을 쓰나?

옛날 HTML 에는 `<b>` (bold), `<i>` (italic) 이 있었습니다.
이 둘은 **오로지 시각적 스타일** 만을 의미했습니다. "여기는 그냥 굵게 그려 주세요" 같은.
그런데 이렇게 쓰면:
- 스크린 리더에게는 아무 의미 없는 정보 (안 읽어 주거나 구분 못함)
- CSS 로 나중에 "굵은 글씨" 스타일을 바꾸려면 `<b>` 의 모양이 어정쩡해짐

그래서 현재는 **의미가 있을 때는 `<strong>`/`<em>` 을, 순수 시각 스타일은 CSS 로** 처리하는 것이 관례예요.
(`<b>`, `<i>` 가 여전히 있긴 하지만 요즘은 거의 안 씁니다.)

---

## 🧱 아무 의미 없는 박스: `<span>`

글자 일부만 **CSS 로 스타일을 바꾸고 싶은데 마땅한 의미 태그가 없을 때** 쓰는 중립 태그입니다.

```html
<p>
  오늘 날씨는 <span class="highlight">정말</span> 좋네요.
</p>
```

`<span>` 은 "나는 아무 의미도 없어요, 그냥 스타일만 바꾸려고 존재하는 그릇이에요" 라는 태그입니다.

### `<span>` vs `<strong>`

| 의미가 있나? | 쓰는 태그 |
|---|---|
| 중요한 강조 | `<strong>` |
| 말의 뉘앙스 강조 | `<em>` |
| 아무 의미 없이 그냥 색만 바꾸고 싶다 | `<span>` + CSS |

> 💡 원칙: **의미 태그가 있으면 의미 태그를 먼저 고려** → 없으면 `<span>` + CSS.

## 🎯 [templates/index.html](../templates/index.html) 에서 찾아 보기

이 레포의 템플릿에서 텍스트 관련 태그들을 확인해 봅시다.

```html
<h1 class="gnb__title text-title">점심 뭐 먹지</h1>
<h2 class="modal-title text-title">음식점 이름</h2>
<h3 class="restaurant__name text-subtitle">피양콩할마니</h3>
<p class="restaurant__description text-body">평양 출신의 할머니가...</p>
```

- 페이지의 대표 제목(상단바) → `<h1>`
- 모달의 제목 → `<h2>`
- 개별 음식점 이름 → `<h3>`
- 음식점 설명 → `<p>`

**제목이 계층적으로** 쓰여 있다는 점에 주목하세요. 이게 좋은 HTML 의 특징입니다.
`class="..."` 는 지금은 신경 쓰지 않아도 됩니다. CSS 를 배울 때 이해하게 될 거예요.

## 🧪 지금 당장 해보기

`02-text.html` 파일을 만들고 써 보세요.

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="utf-8" />
    <title>텍스트 연습</title>
  </head>
  <body>
    <h1>내가 만드는 블로그</h1>

    <h2>React 공부 시작</h2>
    <p>
      오늘부터 React 를 공부한다.
      처음엔 <strong>HTML</strong> 부터 배우기로 했다.
    </p>
    <p>
      생각보다 <em>재미있다.</em>
      점점 <span>감이 잡히는</span> 기분이다.
    </p>

    <hr>

    <h2>내일 할 일</h2>
    <p>
      CSS 공부 시작하기.<br>
      flexbox 가 뭔지 알아보기.
    </p>
  </body>
</html>
```

### 확인해 볼 것

- `<h1>`, `<h2>` 의 크기 차이
- `<strong>` 이 굵게, `<em>` 이 기울여 그려지는지
- `<hr>` 이 가로선으로 나오는지
- `<br>` 이 줄바꿈만 만드는지

### 도전 과제

1. 이 페이지에 **세 번째 `<h2>` 섹션** 을 추가해 보세요.
2. 그 섹션에 `<h3>` 을 2개 넣고, 각 `<h3>` 밑에 `<p>` 를 넣어 "소제목 - 설명" 구조를 만들어 보세요.
3. 중요한 단어 하나를 `<strong>` 으로 감싸 보세요.

## 📚 정리

| 태그 | 의미 | 기본 생김새 |
|---|---|---|
| `<h1>`~`<h6>` | 제목 (중요도 순서) | 크고 굵게 |
| `<p>` | 문단 | 위아래 여백 자동 |
| `<br>` | 줄바꿈 | 한 줄만 내림 |
| `<hr>` | 주제 전환 구분선 | 가로선 |
| `<strong>` | 중요한 강조 | 굵게 |
| `<em>` | 말의 강조 | 기울여서 |
| `<span>` | 의미 없음, CSS 용 그릇 | 평범 |
