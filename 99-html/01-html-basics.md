# 01. HTML 기본 구조와 용어

## 🤔 HTML 은 대체 뭘 하는 물건일까?

브라우저를 열면 네이버, 구글, 유튜브처럼 글자와 이미지와 버튼이 보입니다.
그 화면은 **어디서 온** 걸까요?
답은 단순합니다. **누군가가 작성한 HTML 파일**을 브라우저가 받아서 읽고 그린 것입니다.

HTML 은 **Hyper Text Markup Language** 의 줄임말이에요. 말이 어려우니 풀어서 보면:

- **Hyper Text** → 그냥 "여기저기 링크로 연결된 글" 정도의 뜻.
- **Markup** → 글에 "이건 제목이야", "이건 문단이야", "이건 버튼이야" 하고 **꼬리표를 붙이는 것**.
- **Language** → 그 꼬리표 붙이는 방식의 규칙.

즉 HTML 은 **"브라우저야, 이 글의 이 부분은 제목이고, 이 부분은 버튼이야" 라고 꼬리표를 붙이는 언어** 입니다.

### 꼬리표가 왜 필요해요?

원시적인 텍스트 파일을 하나 떠올려 봅시다.

```
점심 뭐 먹지
오늘 점심은 한식이 좋을 것 같아요.
추가하기
```

이걸 브라우저가 그대로 보여준다면, 브라우저는 다음 중 무엇이 **제목** 이고
무엇이 **설명** 이고 무엇이 **버튼** 인지 전혀 모릅니다.
전부 그냥 같은 모양의 글자로만 찍힐 뿐이죠.

HTML 은 이 문제를 해결하려고 태어났습니다.

```html
<h1>점심 뭐 먹지</h1>
<p>오늘 점심은 한식이 좋을 것 같아요.</p>
<button>추가하기</button>
```

이제 브라우저는 압니다. "아, 첫 줄은 **큰 제목(h1)** 이니까 크고 굵게 그리고,
두 번째 줄은 **문단(p)** 이니까 평범하게 그리고,
세 번째 줄은 **버튼(button)** 이니까 누를 수 있는 모양으로 그려야겠다."

---

## 📦 HTML 의 가장 작은 단위: Tag, Element, Attribute

이 세 단어는 평생 쓰니까 **지금 확실히 구분해 둡시다.**

```html
<a href="https://react.dev">React 홈으로 가기</a>
```

위 코드에서:

### Tag (태그)

`<a>` 와 `</a>` 처럼 꺾쇠로 감싼 표식을 말합니다.
앞에 있는 `<a>` 를 **여는 태그(opening tag)**,
뒤에 있는 `</a>` 를 **닫는 태그(closing tag)** 라고 부릅니다.

### Element (요소)

**여는 태그 + 내용 + 닫는 태그** 를 합친 하나의 덩어리를 가리킵니다.
위 코드 전체(`<a href="...">React 홈으로 가기</a>`) 가 하나의 "a element" 입니다.
"태그와 element 가 뭐가 다르냐" 는 질문을 많이 받는데,

- **태그** 는 "표식" 자체.
- **element** 는 "그 표식으로 감싼 실체 하나" 전체.

라고 생각하면 편해요. 사람들이 일상적으로는 둘을 섞어서 말하기도 합니다.

### Attribute (속성)

여는 태그 안에 `이름="값"` 형태로 들어있는 추가 정보입니다.
위 코드에서 `href="https://react.dev"` 가 attribute 예요.
`href` 가 **이름**, `"https://react.dev"` 가 **값** 입니다.

**같은 태그라도 attribute 로 동작을 다르게** 만들 수 있습니다.
예를 들어 `<a>` 는 **링크** 인데, 어디로 가는 링크인지는 `href` 로 결정돼요.

> 📌 Quick Start 에서 봤던 `<img src="..." alt="..." />` 의 `src`, `alt` 도 전부 attribute 입니다.
> React 로 넘어가면 `<MyButton color="red">` 같은 건 attribute 가 아니라 **props** 라고 부르지만,
> 여는 태그 안에 `이름="값"` 으로 쓰는 생김새는 똑같습니다.

---

## 🧩 닫지 않아도 되는 태그도 있어요

대부분의 태그는 여는 태그와 닫는 태그가 쌍으로 옵니다.
그런데 **내용을 가질 수 없는 태그** 들은 닫을 필요가 없어요.
이런 태그를 **void element** 또는 **self-closing tag** 라고 부릅니다.

```html
<!-- 이미지 -->
<img src="cat.png" alt="고양이 사진">

<!-- 줄바꿈 -->
<br>

<!-- 구분선 -->
<hr>

<!-- 입력창 -->
<input type="text">
```

HTML 에서는 그냥 `<img>` 만 써도 되고, XHTML/JSX 스타일로 `<img />` 처럼 슬래시를 붙여도 됩니다.
**React 의 JSX 에서는 반드시 `<img />` 처럼 슬래시를 붙여야** 해요. 이건 나중에 다시 볼 부분입니다.

## 🏗 HTML 파일의 필수 뼈대

아무 HTML 파일이나 제일 위부터 이런 뼈대로 시작합니다.

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="utf-8" />
    <title>페이지 제목</title>
  </head>
  <body>
    <!-- 여기에 사용자가 보는 모든 내용이 들어감 -->
  </body>
</html>
```

하나씩 왜 있는지 보겠습니다.

### `<!DOCTYPE html>`

**"이 문서는 현대적인 HTML(=HTML5) 의 규칙을 따른다"** 라고 브라우저에게 선언하는 한 줄.
이게 없으면 브라우저가 "오래된 HTML 인가 보네" 하고 이상한 호환 모드로 동작해서 레이아웃이 깨집니다.
**그냥 모든 HTML 파일 맨 위에 붙인다고 외우면 됩니다.**

### `<html lang="ko">`

파일 전체를 감싸는 최상위 태그. `lang="ko"` 는 "이 페이지 언어는 한국어입니다" 라는 뜻이에요.
- 검색 엔진이 "한국어 페이지네" 하고 분류하고,
- 스크린 리더가 "한국어 발음" 으로 읽어 주고,
- 브라우저의 자동 번역 기능이 동작합니다.

### `<head>` vs `<body>`

HTML 에서 가장 중요한 두 영역입니다.

- **`<head>` 안**: 페이지에 **대한** 정보. 화면에는 **보이지 않습니다.**
  - 페이지 제목(브라우저 탭에 뜨는 것)
  - 사용 언어, 인코딩
  - CSS 파일 연결, JavaScript 파일 연결
  - SEO 용 메타 정보
- **`<body>` 안**: 사용자가 **실제로 보는** 모든 것. 텍스트, 이미지, 버튼, 폼 전부.

> 💡 비유: `<head>` 는 책의 **판권 페이지** 같은 정보(저자, 발행일, ISBN)에 해당하고,
> `<body>` 는 실제 본문입니다.

### `<meta charset="utf-8" />`

"이 파일은 UTF-8 이라는 문자 인코딩으로 쓰였어요" 라고 알려줍니다.
한글·이모지·중국어 등이 깨지지 않고 보이려면 **필수** 입니다.

### `<title>페이지 제목</title>`

브라우저 탭에 표시되고, 북마크 이름으로 쓰이고, 검색 결과에 제목으로 나옵니다.

## 🌳 HTML 은 **트리(tree)** 구조다

태그들은 서로 **포함 관계** 를 맺습니다.

```html
<body>
  <header>
    <h1>점심 뭐 먹지</h1>
  </header>
  <main>
    <p>오늘의 추천 음식점</p>
  </main>
</body>
```

이걸 그림으로 그려 보면:

```
body
├── header
│   └── h1 ("점심 뭐 먹지")
└── main
    └── p ("오늘의 추천 음식점")
```

이걸 **부모(parent) / 자식(child) / 형제(sibling)** 관계로 부릅니다.
- `body` 는 `header` 의 **부모**.
- `header` 와 `main` 은 **형제**.
- `h1` 은 `header` 의 **자식**, `body` 의 **손자(grandchild)**.

이 트리 구조를 브라우저가 내부에서 들고 있는 것을 **DOM (Document Object Model)** 이라고 부릅니다.
나중에 React 를 배울 때 엄청나게 많이 듣게 되는 단어라, **"HTML 이 트리 구조라는 점"** 은 지금 확실히 익혀 두세요.

> 🔑 React 가 하는 일: **JavaScript 로 이 트리(DOM) 를 자동으로 만들고 업데이트해 주는 일** 입니다.
> HTML 은 "정적인 트리를 쓰는 법", React 는 "변하는 트리를 자동으로 관리하는 법" 에 대한 이야기입니다.

## 📚 정리

| 용어 | 뜻 | 예시 |
|---|---|---|
| Tag | 꺾쇠로 감싼 표식 | `<h1>`, `</h1>` |
| Element | 태그 + 내용 + 닫는 태그 하나의 덩어리 | `<h1>안녕</h1>` |
| Attribute | 여는 태그 안의 추가 정보 `이름="값"` | `<a href="...">` 의 `href="..."` |
| void element | 내용 없이 혼자 쓰는 태그 | `<img>`, `<br>`, `<input>` |
| DOM | HTML 트리 전체를 가리키는 이름 | — |
