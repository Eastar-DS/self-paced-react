## Phase A-1 — 우아한 TS 책 8장 정독

책을 직접 읽는 단계라 제가 개입할 건 많지 않아요. 대신 **"어디에 집중해서 읽으면 Phase A-2 ~ 로또가 부드러워지는지"** 포인트만 짚어드릴게요.

### 🎯 8장 읽을 때 눈 크게 뜰 부분 4가지

1. **Props 타입 정의의 두 방식 — `interface` vs `type`, 그리고 언제 뭘 쓰는지**
    - 자주 쓰는 건 `interface`. 유니온/교차가 필요하면 `type`. 팀 컨벤션 있으면 따라가는 게 맞음.
2. **이벤트 타입 — `React.ChangeEvent<HTMLInputElement>`, `React.FormEvent<HTMLFormElement>`**
    - 가장 실수 많은 부분. 로또 Ch3 의 당첨번호 input 에서 바로 써요.
    - `e.target.value` 가 왜 string 인지, 숫자로 바꾸려면 어디서 언제 변환할지도 같이 생각.
3. **`useState` 제네릭 타이밍**
    - `useState("")` 처럼 초기값 있으면 TS 가 추론해줌 → 제네릭 불필요.
    - `useState<Lotto[]>([])`, `useState<string | null>(null)` 처럼 **초기값이 narrow 해서 의도한 타입과 다를 때** 만 명시.
    - "언제 쓰고 언제 생략하는지" 감각이 핵심.
4. **`children` 타입 — `ReactNode` vs `ReactElement` vs `JSX.Element`**
    - Phase A-3 재사용 `<Modal>` 만들 때 바로 써요. 대부분의 경우 **`ReactNode`** 가 정답.
    - 셋의 차이를 구분할 수 있으면 충분.
