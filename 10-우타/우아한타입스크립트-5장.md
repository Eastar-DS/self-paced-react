## 1. 🔴 조건부 타입 `T extends U ? X : Y`

**타입을 값처럼 분기하는 문법.** JS 삼항 연산자와 똑같이 생김.

```ts
type IsString<T> = T extends string ? true : false;
type A = IsString<"hello">;  // true
type B = IsString<123>;      // false
```

### 배민 실전 예시 — 결제 수단 분기

사용자가 어떤 타입(`"card" | "appcard" | "bank"`)을 넘기느냐에 따라 반환 타입이 달라져야 함.

```ts
type PayMethodType<T extends "card" | "appcard" | "bank"> =
  T extends "card" | "appcard" ? Card : Bank;

const useGetRegisteredList = <T extends "card" | "appcard" | "bank">(
  type: T
): UseQueryResult<PayMethodType<T>[]> => {
  // ...
};

const cards = useGetRegisteredList("card");  // UseQueryResult<Card[]>
const banks = useGetRegisteredList("bank");  // UseQueryResult<Bank[]>
```

**유니온 리턴 타입(`Card[] | Bank[]`)의 문제:** 호출한 쪽에서 `Card`인지 `Bank`인지 또 좁혀야 함. 조건부 타입은 **입력에 따라 출력이 확정**됨.

## 2. extends 키워드의 세 가지 얼굴

같은 키워드인데 위치에 따라 의미가 다름.

| 위치 | 의미 |
| --- | --- |
| `interface B extends A` | 타입 **상속/확장** |
| `<T extends Constraint>` | 제네릭 **한정자** (T는 Constraint의 부분집합이어야) |
| `T extends U ? X : Y` | 조건부 타입 (**할당 가능성 검사**) |

```ts
// 1. 확장
interface Animal { name: string }
interface Dog extends Animal { bark(): void }

// 2. 제네릭 제약
function firstChar<T extends string>(s: T): string { return s[0]; }

// 3. 조건부
type ElementType<T> = T extends (infer U)[] ? U : T;
```

## 3. 🔴 infer — 타입 내부에서 타입 추론

**"이 자리에 뭔가가 있는데, 그걸 새 타입 변수로 꺼내줘"** 라는 뜻.

### 기본 사용

```ts
type UnpackPromise<T> = T extends Promise<infer K> ? K : never;

type A = UnpackPromise<Promise<string>>;  // string
type B = UnpackPromise<number>;           // never
```

### 표준 유틸리티도 infer로 만들어짐

```ts
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

type Fn = (a: number, b: string) => boolean;
type R = ReturnType<Fn>;     // boolean
type P = Parameters<Fn>;     // [number, string]
```

### 배민 실전 — Promise 배열에서 내부 타입 뽑기

```ts
type UnpackPromise<T> = T extends Promise<infer K>[] ? K : any;

const promises = [Promise.resolve("Mark"), Promise.resolve(38)];
type Expected = UnpackPromise<typeof promises>;  // string | number
```

> 이스타가 3장에서 본 "배민 메뉴 이름 추출" (`UnpackMenuNames`)도 infer + 재귀로 만들어짐.

## 4. 분배적 조건부 타입 (Distributive)

**제네릭 T가 "네이키드(naked) 타입 파라미터"이고 유니온이면, 조건부가 각 멤버에 분배됨.**

```ts
type ToArray<T> = T extends any ? T[] : never;
type A = ToArray<string | number>;
// (string extends any ? string[] : never) | (number extends any ? number[] : never)
// → string[] | number[]
```

### 분배를 막으려면 `[T]` 튜플로 감쌈

```ts
type ToArrayNoDistribute<T> = [T] extends [any] ? T[] : never;
type B = ToArrayNoDistribute<string | number>;  // (string | number)[]
```

### 배민 활용 예시

`Exclude<T, U>`, `Extract<T, U>`가 분배적 조건부로 구현됨.

```ts
type Exclude<T, U> = T extends U ? never : T;
type Extract<T, U> = T extends U ? T : never;

type Primitive = string | number | boolean | null | undefined;
type Objective = Exclude<unknown, Primitive>;
```

## 5. 템플릿 리터럴 타입

JS 템플릿 리터럴 문법을 타입에도 적용. 반복 유니온을 간결하게.

```ts
type HeadingNumber = 1 | 2 | 3 | 4 | 5 | 6;
type HeaderTag = `h${HeadingNumber}`;
// "h1" | "h2" | "h3" | "h4" | "h5" | "h6"

type Vertical = "top" | "bottom";
type Horizon = "left" | "right";
type Direction = Vertical | `${Vertical}${Capitalize<Horizon>}`;
// "top" | "bottom" | "topLeft" | "topRight" | "bottomLeft" | "bottomRight"
```

### 실전 활용
- 디자인 토큰 이름 (`--color-gray-100`, `--color-gray-200`...)
- API 엔드포인트 (`/api/${resource}/${id}`)
- 이벤트 이름 (`on${Capitalize<EventName>}`)

> **주의:** 유니온 멤버가 너무 많으면 조합 폭발로 TS 컴파일러가 느려짐. 100개 이하 유지.

## 6. 🔴 커스텀 유틸리티 — PickOne

**"객체 속성 중 정확히 하나만 가지게 강제"하는 유틸리티.** 배민에서 자주 쓰이는 패턴.

### 문제 상황

결제 시 `card`나 `account` 중 하나만 있어야 함. 유니온으론 둘 다 포함해도 막지 못함.

```ts
type Card = { card: string };
type Account = { account: string };

function withdraw(type: Card | Account) { }

withdraw({ card: "hyundai" });                     // ✅
withdraw({ account: "hana" });                     // ✅
withdraw({ card: "hyundai", account: "hana" });    // ✅ 🚨 막지 못함 (둘 다 있어도 통과)
```

### 해결: PickOne 구현

```ts
type PickOne<T> = {
  [P in keyof T]: Record<P, T[P]> & Partial<Record<Exclude<keyof T, P>, undefined>>;
}[keyof T];

type CardOrAccount = PickOne<Card & Account>;
// { card: string; account?: undefined } | { card?: undefined; account: string }

function withdraw(type: CardOrAccount) { }

withdraw({ card: "hyundai", account: "hana" });  // 🚨 ERROR
```

### 작동 원리
1. 매핑된 타입 `[P in keyof T]`로 각 키에 대해
2. `Record<P, T[P]>`: 그 키는 **필수**로 가지고
3. `Partial<Record<Exclude<keyof T, P>, undefined>>`: 나머지 키는 **옵셔널 + undefined만 허용**
4. `[keyof T]`로 인덱스 접근해서 유니온으로 뽑아냄

> 레벨2 미션 폼 설계에서 "이 필드 **또는** 저 필드" 같은 요구가 생기면 PickOne을 고려.

## 7. NonNullable 타입과 함수

### 타입 (TS 내장)

```ts
type NonNullable<T> = T extends null | undefined ? never : T;

type A = NonNullable<string | null | undefined>;  // string
```

### 런타임 타입 가드 함수

```ts
function nonNullable<T>(value: T): value is NonNullable<T> {
  return value !== null && value !== undefined;
}
```

### 배민 실전 — Promise.all 후 null 제거

```ts
const shopAdCampaignList = await Promise.all(
  shopList.map((shop) => AdCampaignAPI.operating(shop.shopNo))
);
// 타입: (AdCampaign[] | null)[]

const filtered = shopAdCampaignList.filter(nonNullable);
// 타입: AdCampaign[][]  ← null이 타입 레벨에서도 제거됨!
```

**`.filter(Boolean)` 하면 런타임에선 되지만 타입은 안 좁혀짐.** 타입 가드 함수를 만들어 넘겨야 TS가 이해함.

## 8. 불변 객체 (as const + keyof typeof)

값 객체의 키를 타입으로 제약. 이스타가 3장에서 본 `as const` + `typeof`의 확장.

```ts
const theme = {
  colors: {
    default: "#000",
    mint: "#2AC1BC",
    premium: "#FFA500",
  },
  fontSize: {
    default: "16px",
    small: "14px",
    big: "18px",
  },
} as const;

type ColorType = keyof typeof theme.colors;       // "default" | "mint" | "premium"
type FontSizeType = keyof typeof theme.fontSize;  // "default" | "small" | "big"

interface Props {
  color?: ColorType;
  fontSize?: FontSizeType;
}

const Button = ({ color, fontSize }: Props) => { /* ... */ };

<Button color="mint" />;           // ✅
<Button color="rainbow" />;        // 🚨 ERROR
```

**SSOT 원칙의 대표 적용:** 값에서 타입을 뽑으면 값과 타입이 자동으로 동기화됨.

## 9. 🔴 Record 원시 타입 개선 — PartialRecord

**`Record<string, T>`의 함정:** 모든 string 키에 대해 T를 보장한다고 TS가 믿어버림.

```ts
type Category = string;
interface Food { name: string; /* ... */ }

const foodByCategory: Record<Category, Food[]> = {
  한식: [{ name: "제육덮밥" }],
  일식: [{ name: "초밥" }],
};

foodByCategory["양식"].map(/* ... */);
// 🚨 런타임 TypeError: Cannot read properties of undefined
// ⚠️ 컴파일은 통과함 (TS는 Food[]라고 믿음)
```

### 해결 1: 유니온으로 키 좁히기

```ts
type Category = "한식" | "일식" | "중식" | "양식";
const foodByCategory: Record<Category, Food[]> = { /* 4개 전부 필수 */ };
```

→ 새 카테고리가 추가되면 TS가 바로 경고. 카테고리 수가 유한할 때 최선.

### 해결 2: PartialRecord (카테고리가 동적이면)

```ts
type PartialRecord<K extends string, T> = Partial<Record<K, T>>;

const foodByCategory: PartialRecord<string, Food[]> = {
  한식: [/* ... */],
};

foodByCategory["양식"]?.map(/* ... */);
// ✅ 이제 TS가 undefined 가능성을 알려줌. optional chaining 강제됨.
```

### 해결 3: unknown + 타입 가드

```ts
const foodByCategory: Record<string, unknown> = { /* ... */ };

if (Array.isArray(foodByCategory["양식"])) {
  // 좁혀짐
}
```

| 주제 | 언제 |
| --- | --- |
